## Вариант 15 – Управление библиотекой

Приложение должно содержать данные:
- Пользователь
- Книга
- Выдача

Реализовать API:
- создание пользователя
- поиск пользователя
- добавление книги
- поиск книг
- создание выдачи книги пользователю
- получение списка выданных книг пользователю
- возврат книги

---

## Описание системы

Система управления библиотекой предназначена для учета пользователей, каталога книг и операций выдачи/возврата.  
Пользователи взаимодействуют с системой через веб-интерфейс. Запросы поступают на серверную часть (API), где выполняются проверки прав/валидности, доступности книги, создание выдачи и фиксация возврата.  
Для ускорения поиска по каталогу может использоваться отдельный поисковый индекс. Также система интегрируется с внешними сервисами: авторизация, получение метаданных книги по ISBN и отправка уведомлений.

---

## Основные API системы

Реализуются следующие операции:
- создание пользователя
- поиск пользователя
- добавление книги
- поиск книг
- создание выдачи книги пользователю
- получение списка выданных книг пользователю
- возврат книги

---

## Роли пользователей

В системе предусмотрены следующие роли:

**Читатель (Reader)**  
Зарегистрированный пользователь. Может:
- искать книги
- просматривать список своих выдач

**Библиотекарь (Librarian)**  
Пользователь с расширенными правами. Может:
- добавлять книги
- оформлять выдачу книги читателю
- оформлять возврат книги

---

## Внешние системы

**Identity Provider**  
Внешний провайдер аутентификации/авторизации (OIDC/OAuth2). Используется для входа и проверки токенов.

**ISBN Metadata Provider**  
Внешний источник метаданных по ISBN (название, автор, издательство). Используется при добавлении/обогащении карточки книги.

**Сервис уведомлений**  
Внешний сервис отправки уведомлений (email/SMS/мессенджер). Используется для уведомления читателя о выдаче и сроке возврата.

---

## Архитектура системы

Архитектура системы описана в стиле **C4 Model** с использованием **Structurizr DSL**.  
Описание архитектуры находится в `workspace.dsl`.

## Диаграммы

### C1 — System Context

Диаграмма контекста показывает:
- роли пользователей (читатель, библиотекарь)
- взаимодействие пользователей с системой
- интеграцию системы с внешними сервисами (авторизация, метаданные ISBN, уведомления)

### C2 — Container Diagram

Диаграмма контейнеров показывает основные контейнеры системы:
- Reader Web UI
- Staff Web UI
- API Gateway / BFF
- User Service
- Catalog Service
- Loan Service
- Database
- Search Index (если используется)
- внешние системы (Identity Provider, ISBN Metadata Provider, Сервис уведомлений)

Также на диаграмме отображены технологии контейнеров и протоколы взаимодействия.

### Dynamic Diagram

Динамическая диаграмма показывает архитектурно значимый сценарий:
**Выдача книги пользователю**

Сценарий включает:
1. библиотекарь оформляет выдачу через UI
2. запрос проходит через API Gateway / BFF
3. проверяется токен (Identity Provider)
4. проверяются пользователь и доступность книги
5. создается выдача и обновляется статус книги в БД
6. отправляется уведомление пользователю

---

## Используемые технологии

Основные технологии системы:
- UI — Web UI (React)
- API Gateway / BFF — Java 17 (Spring Boot)
- Backend сервисы — Java 17 (Spring Boot)
- База данных — PostgreSQL
- Поисковый индекс — Elasticsearch (опционально)
- Взаимодействие сервисов — HTTPS / REST
- Доступ к БД — JDBC / SQL
- Авторизация — OIDC / OAuth2
- Внешние интеграции — HTTPS API
